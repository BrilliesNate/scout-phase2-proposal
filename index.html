Phase 2 Overview

Phase 2 builds upon the existing Personal Profile and Core Event foundation to introduce a tiered profile and event system:

Profile Types

  • Personal Profile - Enhanced with connections view, past events, and view toggles
  • Artist Profile - Free tier for musicians, DJs, performers with social/music links
  • Business Profile - Paid subscription for venues, vendors, freelancers, organizers

Event Types

  • Core Event - Simplified free events (removing activities, age restrictions, ticket link)
  • Dynamic Event - Premium events with lineups, itineraries, linked profiles, and spaces

Phase 2: Updates to Existing Features

Phase 2 refines the existing Core Event and Personal Profile to create a cleaner foundation for the new profile and event types.

üéâ
Core Event Updates
Simplified Free Events
  • Remove ticket link functionality
  • Remove activities/schedule section
  • Remove age restrictions
  • Remove extra section
  • Retain all other core event features
  • Keep category, tags, location, date/time
  • Maintain event sharing and visibility controls
üë§
Personal Profile Updates
Enhanced Social Features
  • Connection and friend counts display
  • List view for all connections/friends
  • Search through connections
  • Past events attended view
  • Toggle between card/grid event views
  • Personal profiles keep "connections" model
  • Retain all existing profile features

Phase 2: New Profile Types

Two new profile types expand the platform for different user segments.

üé§
Artist Profile
Free Tier
  • Everything from Personal Profile
  • Social links (Instagram, Twitter, etc.)
  • Music links (Spotify, SoundCloud, Apple Music)
  • Title/designation (DJ, Band, Solo Artist, etc.)
  • Scoutables to describe the artist
  • Followers instead of connections
  • Can be linked to Dynamic Events

Target Users: Musicians, DJs, bands, performers. Free to encourage artists as "influencers" who bring audiences to the platform.

üè¢
Business Profile
Paid Subscription
  • Everything from Personal Profile
  • Followers instead of connections
  • Business location (address, map pin)
  • Website URL
  • Entry prices (for venues)
  • Contact information
  • Includes 4 Dynamic Events per subscription
  • Can be linked to other's Dynamic Events

Target Users: Venues (bars, clubs, restaurants), freelancers (photographers, videographers), vendors (security, catering), brands, event organizers.

Phase 2: Dynamic Events

Dynamic Events are the premium event type, unlocking advanced features for complex, multi-day, multi-stage events. Available to Business Profile subscribers (4 included) or as paid add-ons.

‚ö°
Dynamic Event Features
Paid Event Type
  • Everything from Core Event
  • Features removed from Core moved here (activities, age restrictions, ticket link, extra section)
  • Link Business Profiles for venue
  • Link Business Profiles for vendors (security, catering, sound)
  • Link Artist Profiles for performers
  • Dynamic Lineups/Itineraries:
  • Create named "Spaces" (Main Stage, VIP Tent, etc.)
  • Each Space has itinerary items
  • Items include: timeslot, host (linked profile), scoutables
  • Supports multi-day, multi-stage events
  • Perfect for festivals, conferences, weddings

Dynamic Event Use Cases

üéµ Music Festival

  • Multiple stages as Spaces
  • Artist profiles linked to timeslots
  • Vendor profiles for food, merch
  • Venue profile for location

üéì Conference

  • Rooms as Spaces
  • Speaker profiles per session
  • Sponsor business profiles
  • Multi-day schedule

üíí Wedding

  • Areas as Spaces (ceremony, reception)
  • DJ/band as Artist profile
  • Caterer, photographer as vendors
  • Venue as Business profile

üè¢ Corporate Event

  • Meeting rooms as Spaces
  • Presenter timeslots
  • Catering vendor linked
  • Activities throughout day

Optional Features (Time/Budget Dependent)

These items may be included in Phase 2 depending on time and cost:
  • Interactive Map - Show Business Profile locations (mostly venues) and event locations with daily updates
  • Event Reels - Alternative to event cards for a more engaging browsing experience
  • In-App Tickets - Native ticketing system within the app

Development Timeline

The Phase 2 development is structured across a 12-week (3-month) delivery schedule with clear milestones and deliverables.

Weeks 1-2
Architecture & Database Design
Schema design for new profile types (Artist, Business) and Dynamic Events. Subscription model design. API specification documentation. Development environment setup.
Weeks 3-4
Core Event & Personal Profile Updates
Remove deprecated features from Core Events. Implement connection list view, search, and past events. Add card/grid toggle. Backend API updates.
Weeks 5-7
Artist & Business Profile Development
Database migrations and model implementation. Profile creation flows for both types. Social/music links for Artists. Business details, location, subscription setup. Follower system for both profile types.
Weeks 8-10
Dynamic Event System
Dynamic Event creation with linked profiles. Space/Stage management system. Itinerary builder with timeslots and hosts. Profile linking for venues, vendors, artists. Multi-day event support.
Weeks 11-12
Testing, QA & Deployment
Comprehensive testing across all platforms. Subscription flow testing. Performance optimization. Production deployment and monitoring setup. Documentation and handover.
